| **Данные**                              | **Необходимость защиты**                                                                                                                             | **Инструменты защиты**                                                                                                                                                                                                                              |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **PII (персональные данные)**           | Высокая чувствительность. Утечка этих данных может привести к утрате доверия, юридическим последствиям (например, нарушение ФЗ-152).                 | **Шифрование при хранении (AES-256)**, **Шифрование при передаче (TLS 1.3)**, **Управление доступом на основе ролей (RBAC)**, **DLP (Data Loss Prevention)** для предотвращения утечек, **Аудит действий с данными**.                               |
| **PHI (медицинская информация)**        | Высокая чувствительность. Утечка может привести к юридическим последствиям и репутационным рискам для компании.                                      | **Шифрование при хранении (AES-256)**, **Шифрование при передаче (TLS 1.3)**, **Управление доступом (RBAC/ABAC)**, **Аудит и мониторинг доступа**, **Обезличивание для аналитики** (анонимизация данных для статистики).                            |
| **BIOMETRIC (биометрические данные)**   | Очень высокая чувствительность. Требуется отдельное согласие пациента на обработку таких данных, риски утечек — высокие.                             | **Шифрование при хранении (AES-256)**, **Шифрование при передаче (TLS 1.3)**, **Отдельное согласие пользователя**, **Изолированное хранилище** (например, использование MinIO), **Жесткий контроль доступа**, **Аудит**.                            |
| **FINANCIAL (финансовые данные)**       | Высокая чувствительность. Утечка может привести к финансовым потерям и репутационным рискам.                                                         | **Шифрование при хранении (AES-256)**, **Шифрование при передаче (TLS 1.3)**, **Токенизация карт** (например, использование технологии **PCI-DSS**), **Маскировка данных при экспорте** (например, только последние 4 цифры номера карты), **DLP**. |
| **TMC (товарно-материальные ценности)** | Средняя чувствительность. Утечка этих данных может привести к операционным потерям, но не к значительным последствиям.                               | **Шифрование при хранении (AES-256)**, **Шифрование при передаче (TLS 1.3)**, **Контроль экспорта данных (например, блокировка файлов с тегом TMC при попытке выгрузить в открытый доступ)**.                                                       |
| **ARCHIVE (архивные данные)**           | Низкая чувствительность. Однако для архивных данных, содержащих личную информацию, также требуется защита, но последствия утечек не так значительны. | **Шифрование при хранении (AES-256)**, **Автоматическая архивация в защищённые хранилища** (например, через планировщик задач **cron** или **AWS S3** с шифрованием).                                                                               |
| **ANONYMIZED (обезличенные данные)**    | Низкая чувствительность. Обезличенные данные можно использовать для анализа и статистики, не раскрывая личную информацию.                            | **Обезличивание данных** (например, удаление или маскировка идентификаторов), **Шифрование при хранении**, **Контроль экспорта** (например, запрет на возврат идентификаторов).                                                                     |
